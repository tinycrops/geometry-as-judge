story-01

Deep technical dive — cross-modal fidelity evaluation. Text descriptions to ASCII art in Gemini's unified embedding space. Part of ascii-bench.

Can an embedding model judge whether ASCII art faithfully depicts a narrative scene? story-01 tests the hardest variant of the "Geometry as Judge" thesis: cross-modal semantic correspondence between natural language and monospace character art.

The Evaluation

The story-01 module evaluates a fundamentally cross-modal task: given a short story excerpt describing a character or scene, generate ASCII art that depicts it. Unlike computer-01 (which evaluates text-to-text fidelity of terminal command output) or noise-01 (which evaluates corruption tolerance), story-01 asks whether the embedding space captures semantic correspondence across modalities — from natural language description to visual ASCII rendering.

This is the core stress test for the "Geometry as Judge" thesis. If cosine similarity in Gemini's unified embedding space can distinguish good ASCII depictions of story scenes from bad ones, then the space genuinely encodes cross-modal meaning — not just surface-level text similarity.

  • Task: Given a short story excerpt describing a character or scene, generate ASCII art depicting it
  • Dataset: 18 story excerpts with gold ASCII art (12 train, 6 test)
  • Test subjects: mermaid, eagle, train, mushroom, astronaut, crown
  • Metric: Combined text-to-text and text-to-image embedding similarity

Cross-Modal Scoring

The story-01 metric combines two embedding similarity signals with equal weight (0.5 + 0.5):

Signal 1: Text-to-Text

cosine(embed_text(description), embed_text(art_characters)) — does the raw ASCII text carry semantic signal about the subject? This exploits the fact that well-crafted ASCII art often contains text-level hints (character choices, spatial patterns) that correlate with the subject being depicted.

Signal 2: Text-to-Image

cosine(embed_text(description), embed_image(rendered_art_png)) — does the rendered art look like what the text describes? The ASCII art is rendered to a PNG using a monospace font, then embedded via Gemini's image embedding endpoint. This tests whether the visual form of the art maps to the same region of embedding space as the textual description.

Normalization

  • Text-to-text: raw cosine in [0.3, 0.8] mapped linearly to [0, 1]
  • Text-to-image: raw cosine in [0.2, 0.6] mapped linearly to [0, 1]

These ranges were established empirically in Experiment 2 (multimodal metric). The wider normalization range for text-to-image reflects the lower absolute similarity values typical of cross-modal comparisons. Equal weighting (rather than text-to-text-dominant) was chosen deliberately to stress-test both channels.

StoryArtMetric Implementation

class StoryArtMetric:
    def __init__(self, threshold=0.45):
        self.threshold = threshold

    def __call__(self, example, pred, trace=None):
        description = str(getattr(example, "excerpt", ""))
        art = str(getattr(pred, "ascii_art", ""))
        if not art.strip() or not description.strip():
            score = 0.0
        else:
            try:
                desc_vec = embed_text(description)
                art_text_vec = embed_text(art)
                text_sim = cosine(desc_vec, art_text_vec)
                art_png = render(art)
                art_img_vec = embed_image(art_png)
                img_sim = cosine(desc_vec, art_img_vec)
                text_norm = np.clip((text_sim - 0.3) / (0.8 - 0.3), 0.0, 1.0)
                img_norm = np.clip((img_sim - 0.2) / (0.6 - 0.2), 0.0, 1.0)
                score = 0.5 * text_norm + 0.5 * img_norm
            except Exception as e:
                score = 0.0
        if trace is not None:
            return score >= self.threshold
        return score

The dual-mode return (boolean during tracing, float otherwise) follows DSPy convention: during BootstrapFewShot compilation, the metric acts as a gate (pass/fail at threshold 0.45); during evaluation, it returns the continuous score for fine-grained comparison.

Dataset Design

The 18 narrative excerpts span diverse genres and visual complexity levels, deliberately chosen to probe different aspects of cross-modal fidelity:

  • Fantasy: wizard, dragon, castle — complex scenes with multiple visual elements
  • Nature: pine tree, mushroom, flower — distinctive organic shapes
  • Technology: robot, rocket — geometric, structured forms
  • Maritime: sailboat, pirate ship — compound objects with contextual elements (water, sails)
  • Characters: mermaid, astronaut, knight — human-figure variants
  • Objects: crown, hourglass, lighthouse — iconic shapes

Each excerpt is 1-2 sentences with strong visual imagery. Gold ASCII art is 5-10 lines, hand-crafted, focusing on the most distinctive visual features of each subject.

DSPy Program: StoryArtGenerator

The generator uses DSPy's ChainOfThought to reason about which visual features from the story excerpt are most important for ASCII depiction, then produce the art.

class StoryArtGenerator(dspy.Module):
    def __init__(self):
        super().__init__()
        self.generate = dspy.ChainOfThought(
            "excerpt, subject -> ascii_art"
        )

    def forward(self, excerpt, subject):
        result = self.generate(
            excerpt=excerpt,
            subject=subject,
            instructions=(
                "Create ASCII art depicting the subject described in the excerpt. "
                "Focus on the most visually distinctive features mentioned in the text. "
                "Use 5-10 lines. Make the subject immediately recognizable. "
                "Prioritize iconic silhouette and key details over fine ornamentation."
            ),
        )
        return dspy.Prediction(ascii_art=result.ascii_art)

The instructions emphasize recognizable visuals and distinctive features because the metric rewards semantic correspondence. Art that captures the iconic shape of the subject (mushroom cap, eagle wings) will score higher than art that attempts fine detail but loses the overall form.

Key Observations

  • Mushroom scores highest at baseline (0.355). This correctly reflects that subjects with distinctive, simple shapes — round cap on a stem — are easiest to represent in ASCII art and easiest for the embedding model to match to their textual description.
  • Mermaid scores lowest (0.081). An abstract, composite concept (human torso + fish tail) is difficult both to draw in ASCII and for the embedding space to match cross-modally. The low score reflects genuine difficulty, not metric failure.
  • Astronaut shows +111% improvement after optimization. BootstrapFewShot found effective demonstrations for human-figure art with distinctive gear (helmet, suit), allowing the optimized program to produce dramatically better astronaut depictions.
  • Mushroom and crown regressed slightly. This is consistent with optimization trade-offs: the bootstrap demonstrations that helped astronaut and train may have slightly diluted the prompt for subjects that were already well-handled.
  • Overall +3.8% improvement validates that the metric provides useful optimization signal even for cross-modal tasks, though the gains are more modest than computer-01's same-modality improvements.
Why Scores Are Lower Than computer-01

story-01 scores (18-22% range) are substantially lower than computer-01 scores. This is expected for three reasons:

  • Cross-modal evaluation is fundamentally harder. The text-to-image channel requires the embedding model to bridge natural language and rendered monospace character art — two very different modalities.
  • The text-to-image channel contributes lower absolute similarity than text-to-text, even after normalization. Cross-modal cosine similarity in Gemini's space is inherently lower than within-modality similarity.
  • ASCII art is a lossy medium. A 5-10 line character rendering cannot capture the full semantic content of a narrative description. Some information loss is unavoidable.
Discovery

The metric correctly captures subject recognizability in a way that matches human intuition:

  • Distinctive shapes (mushroom: round cap + stem) > complex scenes (train: multiple cars, tracks, smoke) > abstract concepts (mermaid: human-fish hybrid)

This ordering is exactly what we would expect if the embedding space genuinely encodes visual semantics rather than surface-level text overlap. A mushroom looks like a mushroom even in ASCII; a mermaid requires understanding a concept that does not map cleanly to simple character patterns.

The +111% astronaut improvement further demonstrates that optimization CAN produce dramatic gains on specific subjects — particularly those where the baseline struggled not due to inherent difficulty but due to lack of effective prompting strategy for human-figure art with distinctive visual markers.

Open Questions

  • Interleaved embedding: Would embedding text and image together (as a single multimodal input) perform better than computing separate text-to-text and text-to-image similarities? Gemini supports multimodal input — this could capture cross-modal correspondence more directly.
  • Gold reference sensitivity: How does the choice of gold ASCII art affect scores? The current hand-crafted references may not represent the optimal depiction for embedding similarity. Would LLM-generated references — potentially more aligned with the embedding model's learned representations — produce better calibration?
  • Drawability discovery: Can we use the metric in reverse — scoring many candidate subjects against their descriptions — to discover which story elements are most "drawable" in ASCII? This could inform both dataset design and practical applications.
  • Per-channel diagnostics: Breaking out text-to-text and text-to-image scores separately per subject could reveal whether optimization gains come from better textual structure, better visual form, or both.
